# Emplea.do

## Scope
Este es el scope de la plataforma por los próximos meses
- Trabajos de tecnología (from techies to techies)
- Orientado a países Hispanoparlantes
- Facilitar la distinción entre empleos presenciales vs remotos

## Colaboraciones
Hay varias formas de colaborar con el desarrollo de la plataforma, recuerda que todas las colaboraciones deben caer en linea con el scope del proyecto.

### Pull requests (PR)
Clona el repositorio, resuelve un issue y crea un pull request. Antes de hacer merge a todos los pull requests se les hace [code review](https://smartbear.com/learn/code-review/what-is-code-review/). A menos que tu PR sea de un feature hotfix o muy importante, deberás esperar al menos 2 días o a que por lo menos 3 personas revisen tu código antes de que se le haga merge.

### Code reviews

La mejor parte de trabajar en comunidad es que otras personas verán tu código y si ven algo mal te corregirán para que suckees menos. Antes de hacer merge de los pull requests al menos tres personas deben dar el visto bueno de lo que quieres subir. Tu también puedes hacer codereview, solo tienes que ir a un PR, leer el código y escribir comentarios donde veas cosas que te llamen la atención. Si todo está bien agrega un comentario como: "Todo bien", "Cowabunga", "+1", ...

### Issues
Entra a [emplea.do](emplea.do) y si ves algo que se pueda mejorar no tengas miedo de crear un [issue](https://github.com/developersdo/empleo-dot-net/issues). Antes de crearlo revisa que no exista otro similar. Si existe un issue similar agrega tu sugerencia como comentario en ese issue.
